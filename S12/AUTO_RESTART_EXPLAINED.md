# 자동 종료 & 재시작 설명

## 🔄 작동 방식

### 현재 설정
```
작업 스케줄러: 평일 08:00마다 자동 시작
```

### 일일 사이클
```
08:00 → 작업 스케줄러가 모니터링 시작
  ↓
08:00~20:00 → 정상 모니터링 (매 1분마다 체크)
  ↓
20:00 → 자동 종료 (불필요한 대기 제거)
  ↓
20:00~08:00 → 대기 (스케줄러가 시작 안 함)
  ↓
다음날 08:00 → 다시 자동 시작! 🔄
```

## ✅ 확인

TaskName: S2_Realtime_Monitor

## 🎯 왜 자동 종료가 좋은가?

### 이전 방식 (20:00 이후 계속 체크)
```
20:00 → 모니터링 종료되지만...
20:01, 20:02, 20:03... → 계속 "시간 외" 체크
23:59까지 1440회 불필요한 체크
→ CPU 약간 사용, 로그만 증가
```

### 개선된 방식 (자동 종료)
```
20:00 → 즉시 종료
20:00~08:00 → 완전 휴면
→ CPU 사용 0%, 로그 깔끔
다음날 08:00 → 스케줄러가 다시 시작
```

## 🔒 안정성

- **로깅**: 시작/종료 자동 기록
- **복구**: 크래시 시 자동 재시작
