# 실시간 감시 프로그램 모니터링 대시보드 사용 가이드

## 개요

여러 개의 실시간 감시 프로그램들의 실행 상태를 한눈에 확인하고 제어할 수 있는 대시보드입니다.

## 주요 기능

- ✅ **프로세스 상태 확인**: 각 프로그램의 실행 여부 및 PID 표시
- ✅ **로그 상태 모니터링**: 마지막 로그 업데이트 시간 확인
- ✅ **원클릭 시작**: 정지된 프로그램을 즉시 실행
- ✅ **원클릭 정지**: 실행중인 프로그램을 안전하게 종료
- ✅ **일괄 제어**: 모든 프로그램을 한번에 시작/정지

## 실행 방법

### 방법 1: 배치 파일로 실행 (권장)
```bash
run_dashboard.bat
```

### 방법 2: Python 직접 실행
```bash
python monitor_dashboard.py
```

## 대시보드 화면 구성

```
================================================================================
🖥️  실시간 감시 프로그램 모니터링 대시보드
================================================================================
⏰ 업데이트: 2025-11-05 19:14:27
================================================================================

1. S12 주식 실시간 모니터
   상태: 🟢 실행중 (PID: 12345)
   설명: 거래일 08:00-20:00, 10분 간격
   로그: 🟢 30초 전

2. 업비트 시장 하락 감시
   상태: 🔴 정지
   설명: 평일 09:00-18:00, 30분 간격
   로그: 🟠 15:30

3. 암호화폐 실시간 모니터
   상태: 🟢 실행중 (PID: 67890)
   설명: 00:00 파일생성, 30분 간격 알람
   로그: 🟡 3분 전

================================================================================
명령어:
  [1-9]    : 해당 번호 프로그램 시작/재시작
  s[1-9]   : 해당 번호 프로그램 정지
  a        : 모든 프로그램 시작
  x        : 모든 프로그램 정지
  r        : 화면 새로고침
  q        : 종료
================================================================================
```

## 상태 표시 설명

### 프로세스 상태
- 🟢 **실행중**: 프로그램이 정상 실행 중
- 🔴 **정지**: 프로그램이 실행되지 않음
- ⚪ **비활성화**: 모니터링 대상에서 제외됨

### 로그 상태
- 🟢 **활성** (5분 이내): 로그가 최근에 업데이트됨 (프로그램이 활발히 동작 중)
- 🟡 **정상** (1시간 이내): 로그가 비교적 최근에 업데이트됨
- 🟠 **오래됨** (1시간 이상): 로그가 오래전에 업데이트됨 (프로그램이 멈췄을 가능성)
- ❌ **로그 없음**: 로그 파일이 없음 (프로그램이 한번도 실행되지 않음)
- ❓ **확인 불가**: 로그 파일 상태를 확인할 수 없음

## 명령어 사용법

### 개별 프로그램 제어

#### 프로그램 시작
```
명령 입력: 1
```
- 정지된 프로그램을 시작합니다.
- 이미 실행중인 경우, 재시작 여부를 묻습니다.

#### 프로그램 정지
```
명령 입력: s1
```
- 실행중인 프로그램을 강제 종료합니다.

### 일괄 제어

#### 모든 프로그램 시작
```
명령 입력: a
```
- 현재 정지된 모든 프로그램을 한번에 시작합니다.
- 이미 실행중인 프로그램은 건너뜁니다.

#### 모든 프로그램 정지
```
명령 입력: x
```
- 현재 실행중인 모든 프로그램을 한번에 종료합니다.

### 기타 명령

#### 화면 새로고침
```
명령 입력: r
```
- 대시보드를 새로고침하여 최신 상태를 확인합니다.

#### 종료
```
명령 입력: q
```
- 대시보드를 종료합니다. (실행중인 프로그램들은 계속 동작)

## 모니터링 대상 프로그램

현재 대시보드에서 모니터링하는 프로그램들:

### 1. S12 주식 실시간 모니터
- **파일**: `Real_Time_Monitor.py`
- **실행**: `run_real_time_monitor.bat`
- **설명**: 거래일 08:00-20:00 동안 10분 간격으로 Summary 탭 종목 모니터링
- **로그**: `realtime_monitor_YYYYMMDD.log`

### 2. 업비트 시장 하락 감시
- **파일**: `upbit_alert_optimized.py`
- **실행**: `run_upbit_optimized.bat`
- **설명**: 평일 09:00-18:00 동안 30분 간격으로 15개 이상 15% 이상 하락 감시
- **로그**: `upbit_alert_*.log`

### 3. 암호화폐 실시간 모니터
- **파일**: `crypto_realtime_monitor.py`
- **실행**: Python 직접 실행
- **설명**: 00:00에 파일 생성, 30분 간격으로 실시간 가격과 비교하여 알람
- **로그**: `crypto_monitor_*.log`

## 사용 시나리오

### 시나리오 1: 시스템 재부팅 후 모든 프로그램 재시작
```
1. run_dashboard.bat 실행
2. 대시보드에서 모든 프로그램이 🔴 정지 상태인지 확인
3. 명령 입력: a (모든 프로그램 시작)
4. 명령 입력: r (새로고침으로 상태 확인)
5. 모든 프로그램이 🟢 실행중으로 변경되었는지 확인
6. 명령 입력: q (종료)
```

### 시나리오 2: 특정 프로그램만 재시작
```
1. run_dashboard.bat 실행
2. 재시작할 프로그램 번호 확인 (예: 2번 업비트 시장 하락 감시)
3. 명령 입력: s2 (정지)
4. 명령 입력: 2 (시작)
5. 로그 상태가 🟢으로 변경되는지 확인
6. 명령 입력: q (종료)
```

### 시나리오 3: 정기적인 상태 점검
```
1. run_dashboard.bat 실행
2. 각 프로그램의 상태 확인:
   - 프로세스 상태: 🟢인지 확인
   - 로그 상태: 🟢 또는 🟡인지 확인
3. 🔴 정지 상태나 🟠 오래된 로그가 있으면 재시작
4. 명령 입력: q (종료)
```

### 시나리오 4: 실수로 프로그램을 껐을 때
```
1. run_dashboard.bat 실행
2. 🔴 정지 상태인 프로그램 확인
3. 해당 번호 입력하여 즉시 재시작
4. 명령 입력: q (종료)
```

## 프로그램 추가/삭제/수정

`monitor_dashboard.py` 파일의 `MONITORED_PROGRAMS` 리스트를 수정하세요:

```python
MONITORED_PROGRAMS = [
    {
        "name": "프로그램 이름",
        "script": "script_file.py",
        "bat_file": "run_script.bat",  # 없으면 None
        "log_pattern": "log_file_*.log",
        "description": "프로그램 설명",
        "enabled": True  # False로 설정하면 비활성화
    },
    # ... 추가 프로그램
]
```

## 문제 해결

### 프로그램이 시작되지 않을 때
1. bat 파일이 올바른 경로에 있는지 확인
2. Python 경로가 정확한지 확인 (`C:\Program Files (x86)\Python311\python.exe`)
3. 필요한 환경변수나 API 키가 설정되어 있는지 확인

### 프로세스를 찾지 못할 때
- Windows의 `tasklist`와 `wmic` 명령을 사용하여 프로세스를 찾습니다
- 관리자 권한으로 실행해보세요

### 로그 파일이 표시되지 않을 때
- 로그 파일 패턴이 올바른지 확인
- 로그 파일이 스크립트와 같은 디렉토리에 생성되는지 확인

## 주의사항

⚠️ **중요**:
- 대시보드를 종료해도 실행중인 프로그램은 계속 동작합니다.
- 프로그램을 정지(`s` 명령)하면 즉시 강제 종료됩니다. 데이터 손실에 주의하세요.
- 대시보드는 실시간 업데이트가 아닙니다. 최신 상태를 보려면 `r` 명령으로 새로고침하세요.
