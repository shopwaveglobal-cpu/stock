# 표시 모드 상태 설명

## 📺 현재 상황

### 작업 스케줄러 설정
- ✅ `S2_Realtime_Monitor` - 평일 08:00 실행
- ⚠️ 기본적으로 **백그라운드 실행** (화면에 표시 안 됨)
- ⚠️ 스케줄러는 보안상 화면 표시 제한

## 🎬 화면 표시 방식

### 방식 1: 스케줄러 + 별도 모니터 창 (권장)
```
08:00: 스케줄러가 모니터링 시작 (백그라운드)
       watch_logs.ps1 자동 실행 → 별도 창에 로그 표시
```

### 방식 2: 수동 로그 모니터 (실용적)
```
08:00: 모니터링 자동 시작 (백그라운드)
사용자: watch_logs_lite.bat 실행 (필요시)
→ 로그 확인 후 닫기
```

### 방식 3: Telegram 알림 (무인 운영)
```
08:00: 모니터링 자동 시작 (백그라운드)
→ 알림만 받고 로그 안 봄
```

## 💡 추천

**방식 2 + 3 조합이 최선**

```
평상시: Telegram 알림만 확인
확인 필요: watch_logs_lite.bat
디버깅: watch_logs.ps1 (실시간)
```
