# 로그 모니터링 전력 소비 가이드

## 📊 전력 소비 비교

### 시나리오별 전력 추가 소비

| 방법 | CPU 사용률 | 메모리 | 디스크 I/O | 월 추가 전력 |
|------|-----------|--------|-----------|-------------|
| **무모니터링** (기본) | 0% | 0MB | 낮음 | 0kWh |
| **Lite 모드** (10초마다 확인) | 0.5% | 10MB | 매우 낮음 | ~0.2kWh |
| **실시간 모니터** (3초마다) | 1-2% | 30MB | 낮음 | ~1kWh |
| **진짜 실시간** (1초마다) | 3-5% | 50MB | 중간 | ~3kWh |

### 24시간 모니터링 시 월 전력비 추가
- 실시간 모니터 (3초): **약 150원** (kWh당 100원 기준)
- Lite 모드 (10초): **약 30원**

## ✅ 권장 사용법

### 1. 평상시: 모니터링 안 함
```
- 시스템만 실행: 전력 소비 최소
- Telegram 알림으로 상태 확인
- 문제 발생 시에만 로그 확인
```

### 2. 확인 필요 시: Lite 모드
```batch
watch_logs_lite.bat
```
- 필요할 때만 실행 (1분 이내)
- 전력 영향 거의 없음

### 3. 디버깅 시: 실시간 모니터
```batch
watch_logs.bat
또는
.\watch_logs.ps1 -Type today
```
- 문제 발생 시에만 사용
- 1-2시간 이내로 제한

### 4. 원격 확인: 원격 데스크톱
```batch
실제 PC: 끔
원격 접속: 필요시만
전력 소비: 매우 낮음
```

## 🔌 전력 절약 팁

### 최적 설정
1. **화면 끄기**: 매핑 모니터로 유지
2. **화면 보호기**: 없음 (불필요한 전력)
3. **하드 디스크**: 절전 끄기
4. **USB 장치**: 꺼두기
5. **LED 인디케이터**: 상관 없음

### 원격 데스크톱 권장
```powershell
# 설정 → 시스템 → 원격 데스크톱 → 켜기
```

**장점:**
- 주 모니터 끄기 가능
- 무선 마우스/키보드 끄기 가능
- 실제로 더 전력 절약!

## 📈 실제 사용 예시

### 시나리오 1: 진짜 24/7 무인 운영
```
모니터링: None
전력 추가: 0원
확인 방법: Telegram 알림만
```

### 시나리오 2: 하루 1회 확인
```
모니터링: Lite 모드 1분
전력 추가: 30원/월
확인 방법: Telegram + 오전 1회 확인
```

### 시나리오 3: 개발/테스트 중
```
모니터링: 실시간 2시간
전력 추가: 150원/월
확인 방법: 디버깅 중에만
```

## ⚡ 결론

**전력 관점에서:**
- ✅ **모니터링 안 하는 게 최선**
- ✅ Telegram 알림만으로 충분
- ✅ 문제 발생 시에만 로그 확인
- ✅ 로그 모니터링은 전력 소비 미미함

**실용적 관점에서:**
```
평상시: 시스템만 실행 + Telegram 알림
확인 필요: watch_logs_lite.bat (1분 이내)
디버깅: watch_logs.ps1 (2시간 이내)
```
