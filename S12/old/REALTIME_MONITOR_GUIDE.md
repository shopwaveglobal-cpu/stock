## 🚀 실시간 모니터링 시스템

### 특징
- **모니터링 시간**: 08:00~20:00 (장중 시간대)
- **알람 조건**: 매수선 **5% 이내** 접근 시
- **중복 방지**: 같은 종목 같은 조건은 하루 1회만 알람
- **동적 계산**: 과거 19일 + 현재가로 실시간 20일선 계산

### 작동 방식
1. Summary 탭의 종목들을 순차 스캔
2. 각 종목의 현재가 조회
3. 동적 20일선 및 매수선 계산
4. 5% 이내 접근 시 텔레그램 알람 전송
5. alert_history.json에 기록 (중복 방지)

## ⚙️ 임계값 조정

### Real_Time_Monitor 임계값 변경

코드를 직접 수정해야 합니다:

```python
# Real_Time_Monitor.py 35번째 줄
DISTANCE_THRESHOLD = 5.0  # 5% 이내 접근 시 알람

# 예: 10%로 변경
DISTANCE_THRESHOLD = 10.0  # 10% 이내 접근 시 알람
```

### Trading_Signal_System 임계값 변경

옵션으로 간단하게 변경 가능:

```bash
# 3% 이내만 알람 (강함)

# 10% 이내 알람 (기본)
python Trading_Signal_System.py --appkey [KEY] --secret [SECRET] --alert-threshold 10

# 20% 이내 알람 (약함)
```

## 🔧 문제 해결

### 알람이 안 올 때

**1. 실시간 모니터링 (5% 이내 없음)**
- 5% 임계값이 너무 엄격
- 코드에서 `DISTANCE_THRESHOLD` 값을 10.0으로 변경

**2. 모니터링 시간대 확인**
- 08:00~20:00 사이에만 작동
- 시간대 밖에서는 "모니터링 시간대가 아닙니다" 메시지

**3. Summary 탭이 비어있음**
- `trading_signals.xlsx`의 Summary 탭에 종목이 있어야 함
- Trading_Signal_System을 먼저 실행해서 Summary 생성

### API 토큰 오류

```bash
✗ 접근 토큰 발급 실패
```

- API 키 확인 (`--appkey`, `--secret`)
- 네트워크 연결 확인
- API 서버 상태 확인
