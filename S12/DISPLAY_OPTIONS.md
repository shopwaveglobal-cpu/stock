# 실시간 모니터링 표시 옵션

## 🎯 상황 정리

**요청:** 매일 8시부터 실시간 감시가 시작될 때 자동으로 창이 켜져서 보이게

**문제:** Windows 작업 스케줄러는 기본적으로 백그라운드 실행 (보안상 이유)

## ✅ 해결 방법

### 방법 1: 수동 창 열기 (추천 ⭐)

실시간 모니터링이 시작된 후 필요할 때만:

```batch
# 간단 버전
watch_logs_lite.bat

# 실시간 업데이트
.\watch_logs.ps1 -Type today
```

**장점:**
- 전력 절약
- 필요할 때만 확인
- 간단함

### 방법 2: 작업 스케줄러 + 트레이 아이콘

작업 스케줄러 실행 → 트레이에 실행 중 표시

### 방법 3: Telegram 알림 (가장 권장!)

**무인 운영의 최선:**
- 로그를 보지 않아도 됨
- Telegram 알림만으로 충분
- 문제 발생 시에만 확인

## 🖥️ 실제 사용 시나리오

### 평상시 (무인 운영)
```
시스템: 백그라운드 실행
확인: Telegram 알림만
표시: 없음
```

### 확인 필요 시
```
watch_logs_lite.bat 실행
→ 1분 안에 확인 후 종료
```

### 디버깅 시
```
.\watch_logs.ps1 -Type today
→ 실시간 업데이트
→ 완료 후 Ctrl+C
```

## 💡 추천

**가장 실용적인 방법:**

1. **매일 오전 9시:** Telegram 리포트 확인
2. **문제 발생 시:** `watch_logs_lite.bat` 실행
3. **그 외:** 로그 확인 불필요

**결론:** 
- 표시 창 자동 열기는 불필요
- Telegram + 필요시 Lite 모드 확인
- 전력 소비도 적고 효율적!

## 🔧 작업 스케줄러 표시 모드 (선택사항)

만약 정말 필요하다면:

1. 작업 스케줄러 열기 (Win+R → taskschd.msc)
2. S2_Realtime_Monitor 작업 찾기
3. 속성 → 일반 → 보안 옵션 → "가장 높은 권한으로 실행" 체크
4. 저장

**주의:** 보안 경고가 뜰 수 있고, 매번 로그인 필요할 수 있음
