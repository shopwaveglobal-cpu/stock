# S1 실시간 감시 알람 조건 상세 분석

## 알람 발생 조건 요약

S1 시스템은 다음과 같은 조건에서 알람을 전송합니다:

### ✅ 매수 체결 알람 (최우선)
- **조건**: 저가가 매수선에 도달하거나 넘어섰을 때 (`low_dist <= 0`)
- **발생 시점**: 실시간 체결 시
- **알람 타입**: 
  - `1차 매수 체결!` ✅
  - `2차 매수 체결!` ✅✅
  - `3차 매수 체결!` ✅✅✅

### ✅ 매수선 인접 알람 (3단계)
- **조건**: 저가가 매수선에 가까워질 때
- **발생 시점**: 실시간 모니터링 중
- **알람 타입**:
  - `1% 인접` 🔴 (0 < 이격도 <= 1%)
  - `3% 인접` 🟠 (1% < 이격도 <= 3%)
  - `5% 인접` 🟡 (3% < 이격도 <= 5%)

---

## 1. 1차 매수선 알람 (매수 상태: NONE)

### 1.1 매수 체결 알람

**조건:**
```python
if low_dist_buy1 <= 0:  # 저가가 매수선에 도달하거나 넘어섬
    alert_type = "1차 매수 체결!"
```

**예시:**
- 1차 매수선: 100,000원
- 저가: 99,500원 (이격도: -0.5%)
- **결과**: ✅ `1차 매수 체결!` 알람 전송

**특징:**
- 매도가 정보 포함 (3% 매도가, 5% 매도가, 7% 매도가)
- 하루에 1회만 전송 (중복 방지)
- 텔레그램 + 슬랙 동시 전송

### 1.2 매수선 인접 알람

#### 🔴 1% 인접 알람

**조건:**
```python
if 0 < low_dist_buy1 <= 1:  # 저가가 매수선에서 1% 이내
    alert_type = "1차 매수선 1% 인접"
```

**예시:**
- 1차 매수선: 100,000원
- 저가: 100,500원 (이격도: 0.5%)
- **결과**: 🔴 `1차 매수선 1% 인접` 알람 전송

#### 🟠 3% 인접 알람

**조건:**
```python
elif 1 < low_dist_buy1 <= 3:  # 저가가 매수선에서 1~3% 사이
    # 단, 1% 알람이 이미 전송된 경우 스킵
    if not ticker_alerts.get("BUY1_1PCT", False):
        alert_type = "1차 매수선 3% 인접"
```

**예시:**
- 1차 매수선: 100,000원
- 저가: 102,000원 (이격도: 2.0%)
- **결과**: 🟠 `1차 매수선 3% 인접` 알람 전송 (1% 알람이 없을 때만)

#### 🟡 5% 인접 알람

**조건:**
```python
elif 3 < low_dist_buy1 <= 5:  # 저가가 매수선에서 3~5% 사이
    # 단, 1% 또는 3% 알람이 이미 전송된 경우 스킵
    if not (ticker_alerts.get("BUY1_1PCT", False) or ticker_alerts.get("BUY1_3PCT", False)):
        alert_type = "1차 매수선 5% 인접"
```

**예시:**
- 1차 매수선: 100,000원
- 저가: 104,000원 (이격도: 4.0%)
- **결과**: 🟡 `1차 매수선 5% 인접` 알람 전송 (1%, 3% 알람이 없을 때만)

**우선순위:**
1. 1% 인접 알람 (최우선)
2. 3% 인접 알람 (1% 알람이 없을 때만)
3. 5% 인접 알람 (1%, 3% 알람이 없을 때만)

## 알람 전송 로직

### 기준 가격

**중요**: 모든 알람은 **저가(low_price)** 기준으로 계산됩니다.

```python
low_dist_buy1 = calculate_low_price_distance(low_price, buy1)
# 저가와 매수선 사이의 이격도 계산
```

**이유:**
- 저가는 당일 최저가를 의미
- 저가가 매수선에 도달했다 = 당일 매수 체결 가능
- 현재가보다 더 정확한 체결 판단 가능

### 중복 방지

**하루 1회 전송 원칙:**
- 각 알람 타입마다 하루에 1회만 전송
- `alert_history.json` 파일에 기록
- 같은 알람이 다시 발생해도 전송하지 않음

**알람 키 (Alert Key):**
- `BUY1_EXECUTED`: 1차 매수 체결
- `BUY1_1PCT`: 1차 매수선 1% 인접
- `BUY1_3PCT`: 1차 매수선 3% 인접
- `BUY1_5PCT`: 1차 매수선 5% 인접
- `BUY2_EXECUTED`: 2차 매수 체결
- `BUY2_1PCT`: 2차 매수선 1% 인접
- `BUY2_3PCT`: 2차 매수선 3% 인접
- `BUY2_5PCT`: 2차 매수선 5% 인접
- `BUY3_EXECUTED`: 3차 매수 체결
- `BUY3_1PCT`: 3차 매수선 1% 인접
- `BUY3_3PCT`: 3차 매수선 3% 인접
- `BUY3_5PCT`: 3차 매수선 5% 인접

### 우선순위

**인접 알람 우선순위:**
1. **1% 인접** (최우선) - 가장 가까울 때
2. **3% 인접** (차순위) - 1% 알람이 없을 때만
3. **5% 인접** (최후) - 1%, 3% 알람이 없을 때만

**매수 체결 알람:**
- 매수 체결 알람은 인접 알람보다 우선
- 저가가 매수선에 도달하면 체결 알람만 전송
